# MindBridge Dashboard Completion Prompt

**Objective**: 
Build out the remaining unfinished sections of the Student Dashboard and the Counselor Dashboard in the MindBridge application. All new components must strictly adhere to the project's existing "Linear-inspired" premium design system, utilizing the established color tokens, layout patterns, and interactive micro-animations.

**Context**: 
The project has recently migrated to a demo-based authentication system using cookie sessions instead of Supabase OAuth. The app has `student`, `counselor`, and `admin` roles. Data is loaded through Next.js server components via the `supabase` and `demo-auth` layers. 

## 1. Student Dashboard Remaining Sections

Currently, the Student Dashboard has basic overviews, but it is missing the deeper interactive pages for:

### A. Detailed Mood History & Analytics Page (`/dashboard/student/mood-history`)
- **Requirements**:
  - Build a responsive layout displaying the student's historical `mood_logs`.
  - Implement a sleek, interactive chart/graph using a minimalist charting library (e.g., Recharts) or custom SVG paths with smooth stroke animations.
  - Apply glassmorphism on tooltips or hover states.
  - Include filtering options (Last 7 Days, Last 30 Days, All Time) using modern segmented controls.

### B. Counselor Booking Flow (`/dashboard/student/book-counselor`)
- **Requirements**:
  - Build an intuitive, grid-based calendar view showing available `counselor_slots`.
  - The UI should feel like a premium booking experience (similar to Cal.com or Linear's date pickers).
  - Include dynamic state transitions: empty states for no availability, hover effects on clickable slots, and loading spinners (using the existing global styles) during the booking mutation.

## 2. Counselor Dashboard Remaining Sections

The Counselor Dashboard needs to be fully fleshed out to allow professionals to manage their students and schedule efficiently:

### A. Student Management & Crisis Alerts (`/dashboard/counselor/students`)
- **Requirements**:
  - Build a data table or a grid of cards displaying assigned students.
  - Integrate visual badges to highlight students who have active or unresolved `crisis_logs` (e.g., a subtle pulsing red dot or badge using `text-red-400 bg-red-400/10`).
  - Create a slide-out panel (drawer) or a modal to view a specific student's recent mood trends and session notes.

### B. Availability & Slot Manager (`/dashboard/counselor/schedule`)
- **Requirements**:
  - Provide a drag-and-drop interface or an intuitive form for counselors to define and manage their weekly `counselor_slots`.
  - Upcoming bookings should be visually distinct from open/unbooked slots.
  - Use our standardized button components and input fields for creating new availability.

## Design System Guidelines to Follow:
- **Colors**: Use the established Tailwind variables (`var(--background)`, `var(--foreground)`, `var(--muted)`, `var(--border)`, etc.). No generic colors.
- **Typography**: Utilize the existing font (Inter/Outfit) with clean hierarchies. Use `tracking-tight` for large headings.
- **Micro-animations**: Ensure all clickable elements (buttons, cards) have `transition-all duration-200 ease-in-out` with hover states like `hover:bg-white/5` or `hover:scale-[1.02]`.
- **Borders & Radii**: Use `rounded-xl` or `rounded-2xl` on cards, combined with a 1px border (`border-white/10`) to achieve the premium outlined look.

**Implementation Workflow**:
1. Review the existing components in `src/components/` and `src/app/globals.css` to understand the established style.
2. Implement the new pages inside `src/app/dashboard/(student|counselor)/...` directories as Next.js App Router server/client components.
3. Ensure server components safely fetch data using `supabase` and pass initial data to client components for interactivity.
